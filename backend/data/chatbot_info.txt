# AI PORTFOLIO ASSISTANT - Technical Documentation

## Overview
This AI assistant was built by Usama Masood as an intelligent portfolio chatbot to answer questions about his professional background, skills, projects, and experience.

## How Usama Built This Chatbot

### Architecture & Design
- **Architecture Type:** RAG (Retrieval-Augmented Generation)
- **Purpose:** Provide accurate, consistent answers about Usama's professional profile
- **Approach:** Hybrid search combining semantic understanding with keyword matching

### Technologies Used

#### Backend Stack
- **Framework:** FastAPI (Python web framework for APIs)
- **Language Model:** Groq API with Llama 3.1 8B Instant model (FREE tier)
- **Vector Database:** FAISS (Facebook AI Similarity Search)
- **Embeddings:** Sentence Transformers (all-MiniLM-L6-v2)
- **LLM Framework:** LangChain (orchestration and chains)
- **Memory:** ConversationBufferMemory (conversation history tracking)

#### Frontend Stack
- **Framework:** React with Vite
- **UI Components:** Custom components with Tailwind CSS
- **Markdown Rendering:** ReactMarkdown with syntax highlighting
- **Voice Features:** Web Speech API for voice input/output

### Key Technical Features

#### 1. RAG Implementation
- **Knowledge Base:** Multiple text files containing Usama's complete professional information
- **Chunking Strategy:** 500 character chunks with 50 character overlap
- **Retrieval:** Top 5 most relevant document chunks per query
- **Embedding Model:** Sentence transformers for semantic search

#### 2. Hybrid Retrieval System
- **Semantic Search (70%):** Vector similarity using FAISS
- **Keyword Search (30%):** BM25 algorithm for exact matches
- **Smart Retriever:** Automatic query classification and routing

#### 3. Conversation Memory
- **Type:** Buffer memory maintaining full conversation history
- **Scope:** Session-based (unique session ID per user)
- **Benefit:** Understands context and follow-up questions

#### 4. Accuracy Optimizations
- **Temperature:** 0.2 (low temperature for consistent, factual answers)
- **Prompt Engineering:** Simple, direct instructions to extract exact facts
- **Consistency:** Same question always produces same answer

#### 5. Free Model Integration
- **Primary Model:** Groq API (free tier, very fast inference)
- **Fallback Option:** HuggingFace Transformers (offline capability)
- **Cost:** $0 - completely free to run

### Technical Specifications

#### Performance
- **Response Time:** 1-3 seconds average
- **Context Window:** 5 relevant document chunks
- **Model Speed:** Groq provides very fast inference (~500 tokens/sec)
- **Caching:** Frequently asked questions cached for instant responses

#### Data Pipeline
1. **Data Loading:** Text files loaded from /data directory
2. **Document Chunking:** Split into manageable pieces
3. **Embedding Generation:** Convert chunks to vectors
4. **Vector Storage:** Store in FAISS index
5. **Retrieval:** Semantic + keyword search on user query
6. **Generation:** LLM generates answer from retrieved context

#### Security & Safety
- **Content Filtering:** Sanitization of user inputs
- **Rate Limiting:** 20 requests per minute per IP
- **Sentiment Analysis:** Tracking user sentiment
- **Safety Checks:** Blocking unsafe or inappropriate content

### Why This Approach?

#### Advantages of RAG
1. **Accuracy:** Answers grounded in actual documents, not hallucinated
2. **Up-to-date:** Knowledge base can be updated without retraining
3. **Transparency:** Can trace answers back to source documents
4. **Cost-effective:** Uses free models with minimal compute
5. **Control:** Full control over knowledge and responses

#### Why Groq?
- Free API tier with generous limits
- Extremely fast inference (faster than OpenAI)
- Llama 3.1 model is highly capable
- No credit card required
- Good for production use

### Key Learnings & Innovations

1. **Simplified Prompt Design:** Removed complex tier systems for better accuracy
2. **Conversation Context:** Maintains history so follow-up questions work naturally
3. **Hybrid Retrieval:** Combining semantic and keyword search improves accuracy
4. **Low Temperature:** Set to 0.2 for consistent, factual responses
5. **Increased Context:** Retrieve 5 documents instead of 3 for better coverage
